## 修复 CI 依赖问题 + Release v0.2.2

- **根因**：`package.json` 的 `devDependencies` 中声明了 `zod`，但代码中从未 import/使用
- `zod` 在 OpenCode CI 的 `bun install --no-cache` 环境下触发 `ENOENT: failed opening cache/package/version dir for package zod`
- **修复**：从 `devDependencies` 移除 `zod`，用 `npm uninstall zod` 确保 lockfile 也清理干净
- 版本升级到 v0.2.2，更新 CHANGELOG
